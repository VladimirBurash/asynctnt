__init__(*, host='127.0.0.1', port=3301, username=None, password=None, fetch_schema=True, auto_refetch_schema=True, connect_timeout=60, request_timeout=-1, reconnect_timeout=0.3333333333333333, tuple_as_dict=False, encoding=None, initial_read_buffer_size=None, loop=None)[source]¶

Connection constructor.

Parameters:
  • host – Tarantool host (pass unix/ to connect to unix socket)
  • port – Tarantool port (pass /path/to/sockfile to connect ot unix socket)
  • username – Username to use for auth (if None you are connected as a guest)
  • password – Password to use for auth
  • fetch_schema – Pass True to be able to use spaces and indexes names in data manipulation routines (default is True)
  • auto_refetch_schema – If set to True then when ER_WRONG_SCHEMA_VERSION error occurs on a request, schema is refetched and the initial request is resent. If set to False then schema will not be checked by Tarantool, so no errors will occur
  • connect_timeout – Time in seconds how long to wait for connecting to socket
  • request_timeout – Request timeout (in seconds) for all requests (by default there is no timeout)
  • reconnect_timeout – Time in seconds to wait before automatic reconnect (set to 0 or None to disable auto reconnect)
  • tuple_as_dict – Bool value indicating whether or not to use spaces schema to decode response tuples by default. You can always change this behaviour in the request itself. Note: fetch_schema must be True
  • encoding – The encoding to use for all strings encoding and decoding (default is utf-8)
  • initial_read_buffer_size – Initial and minimum size of read buffer in bytes. Higher value means less reallocations, but higher memory usage.
  • loop – Asyncio event loop to use

call(func_name, args=None, *, timeout=-1)[source]¶

Call request coroutine. It is a call with a new behaviour (return result of a Tarantool procedure is not wrapped into an extra tuple). If you’re connecting to Tarantool with version < 1.7, then this call method acts like a call16 method

Examples:

# tarantool function:
# function f(...)
#     return ...
# end

>>> res = await conn.call('f')
>>> res.body
[]

>>> res = await conn.call('f', [20, 42])
>>> res.body
[20, 42]
Parameters:
  • func_name – function name to call
  • args – arguments to pass to the function (list object)
  • timeout – Request timeout

insert(space, t, *, replace=False, timeout=-1, tuple_as_dict=None)[source]¶

Insert request coroutine.

Examples:

# Basic usage
>>> res = await conn.insert('tester', [0, 'hello'])
>>> res
<Response: code=0, sync=7, body_len=4>
>>> res.body
[[0, 'hello']]

# Getting dict results
>>> res = await conn.insert('tester', [0, 'hello'],
...                         tuple_as_dict=True)
>>> res.body
[{'id': 0, 'text': 'hello'}]

# Using dict as an argument tuple
>>> res = await conn.insert('tester', {
...                             'id': 0
...                             'text': 'hell0'
...                         },
...                         tuple_as_dict=True)
>>> res.body
[{'id': 0, 'text': 'hello'}]
Parameters:
  • space – space id or space name.
  • t – tuple to insert (list object)
  • replace – performs replace request instead of insert
  • timeout – Request timeout
  • tuple_as_dict – Decode tuple according to schema

update(space, key, operations, **kwargs)[source]¶

Update request coroutine.

Examples:

# Assuming tuple [0, 'hello'] is in space tester

>>> res = await conn.update('tester', [0],
...                         [ ['=', 1, 'hi!'] ])
>>> res.body
[[0, 'hi!']]

# you can use fields names as well
>>> res = await conn.update('tester', [0],
...                         [ ['=', 'text', 'hola'] ])
>>> res.body
[[0, 'hola']]

# ... and retrieve tuples as dicts, of course
>>> res = await conn.update('tester', [0],
...                         [ ['=', 'text', 'hola'] ],
...                         tuple_as_dict=True)
>>> res.body
[{'id': 0, 'text': 'hola'}]
Parameters:
  • space – space id or space name.
  • key – key to update
  • operations – Operations list of the following format: [ [op_type, field_no, …], … ]. Please refer to https://tarantool.org/doc/book/box/box_space.html?highlight=update#lua-function.space_object.update You can use field numbers as well as their names in space format as a field_no (if only fetch_schema is True). If field is unknown then TarantoolSchemaError is raised.
  • index – index id or name
  • timeout – Request timeout
  • tuple_as_dict – Decode tuple according to schema
Returns:

asynctnt.Response instance
upsert(space, t, operations, **kwargs)[source]¶

Update request coroutine. Performs either insert or update (depending of either tuple exists or not)

Examples:

# upsert does not return anything
>>> res = await conn.upsert('tester', [0, 'hello'],
...                         [ ['=', 1, 'hi!'] ])
>>> res.body
[]
Parameters:
  • space – space id or space name.
  • t – tuple to insert if it’s not in space
  • operations – Operations list to use for update if tuple is already in space. It has the same format as in update requets: [ [op_type, field_no, …], … ]. Please refer to https://tarantool.org/doc/book/box/box_space.html?highlight=update#lua-function.space_object.update You can use field numbers as well as their names in space format as a field_no (if only fetch_schema is True). If field is unknown then TarantoolSchemaError is raised.
  • timeout – Request timeout
  • tuple_as_dict – Decode tuple according to schema. Has no effect in upsert requests
Returns:
